Replaced the htmx form-trigger with explicit JS-driven saves: `scheduleHandleSave()` debounces 400 ms on input, `saveHandlesNow()` fires immediately on row-delete, and both use `htmx.ajax('POST', '/api/handles', { source: '#handles-form', swap: 'none' })` which reliably serializes the form regardless of how its fields were added. ‚Äî 2026-04-22
- `src/index.html`: caption preview didn't render when a message was clicked in the list. Same bug existed on the handle-row delete buttons (inline `htmx.trigger` calls) and there was no trigger at all when typing in a handle value ‚Äî both now route through `refreshPreview()`. ### Added (gateway / bot-online presence)
- `src/gateway.rs` (new): minimal hand-rolled Discord Gateway v10 client (`tokio-tungstenite` + `futures-util`). Connects, IDENTIFYs with `intents: 0`, runs the heartbeat loop, reconnects on disconnect with exponential backoff (1 s ‚Üí 60 s cap). Fatal close codes (4004 invalid token, 4010‚Äì4014 invalid shard/intents/API version) stop the loop so a misconfigured deployment doesn't spin forever. Holding this connection is what makes the bot appear **online** in Discord ‚Äî REST alone doesn't affect presence. ‚Äî 2026-04-22
- `Cargo.toml`: added `tokio-tungstenite` 0.24 (`connect` + `rustls-tls-native-roots`) and `futures-util` 0.3 (`std` + `sink`). ‚Äî 2026-04-22
- `src/main.rs`: spawn `gateway::run` on startup alongside the poller when the token is set; new `AppCtx.gateway_connected: Arc<AtomicBool>` + `GET /api/gateway/status` endpoint returns an HTML badge (`online`/`offline`). ‚Äî 2026-04-22

### Changed (gateway-triggered real-time reactions)
- `src/gateway.rs`: IDENTIFY now requests `GUILD_MESSAGES` (`IDENTIFY_INTENTS = 1 << 9`, non-privileged) instead of `intents: 0`. The receive loop now handles `MESSAGE_CREATE`: on a match for `ctx.channel_id`, pushes a log line and calls `ctx.poll_trigger.notify_one()`. `MESSAGE_CONTENT` is deliberately NOT requested ‚Äî it's privileged and unnecessary since we fetch bodies via REST. ‚Äî 2026-04-22
- `src/main.rs`: new `AppCtx.poll_trigger: Arc<tokio::sync::Notify>`, initialized in `main()`, passed to both the gateway spawn and the poller. The poller's inter-fetch sleep now uses `tokio::select!` so the 30 s timer races the trigger ‚Äî reactions happen within a few seconds of a new announcement instead of waiting out the full cycle. The timer remains the safety net if the gateway briefly drops. ‚Äî 2026-04-22
