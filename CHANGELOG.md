### Changed
- **Swapped the desktop GUI for a web server.** The frontend is now a browser UI served by an embedded HTTP server on `PORT` (default 8080). Rationale: simpler docker-compose deployment, no X11/Wayland in-container acrobatics. ‚Äî 2026-04-22
  - `Cargo.toml`: removed `eframe`, `egui_extras`, `image`, `ureq`; added `tokio` (rt-multi-thread, macros, time, sync, signal), `axum` 0.7, `tower-http` 0.6 (fs), `reqwest` 0.12 (rustls).
  - `src/discord.rs`: converted to async. `Client::fetch_messages` and `Client::add_reaction` now take `&self` and return `impl Future`. All four existing parser / encoder tests still pass unchanged.
  - `src/main.rs`: rewritten as a `#[tokio::main]` axum server. Routes: `GET /` (single-page app), `GET /api/config`, `POST /api/fetch`, `POST /api/preview`, `POST /api/poller/toggle`, `GET /api/poller/status`, `GET /api/poller/log`, `GET /images/*` (static files via `ServeDir`). Graceful shutdown on SIGINT.
  - `src/main.rs`: poller ported from `std::thread` to `tokio::spawn` using the same `Arc<AtomicBool>` stop flag pattern and a `tokio::sync::Mutex<VecDeque>` for the shared log.
  - `src/index.html` (new): single-page UI built on htmx 2 (CDN-loaded). Dark theme, four-panel layout (messages / convert / image / handles) plus a bottom log tail polled every 3 s. Character counter turns red past 2,200 (Instagram's cap). Clicking a message in the list fills the textarea and auto-seeds the handle table with its mention IDs.
  - Security note: **the web UI has no authentication.** Bot token stays env-only ‚Äî there's no UI field to read or modify it. Bind the listener to localhost or a trusted docker network.
  - Smoke-tested: `PORT=18080 cargo run` boots, `GET /` returns 10 KB HTML, `GET /api/config` returns the expected JSON, `GET /api/poller/status` returns the right HTML fragment. Real Discord API paths untested locally (no token). ‚Äî 2026-04-22
